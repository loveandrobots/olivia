- **20:51** — Tenth heartbeat: woke for OLI-175 (assigned). Board configured all 7 GH secrets, but CI failed with `Failed to load AuthKey file`. Root cause: `xcrun altool --apiKeyPath` expects a **directory** containing `AuthKey_{KEY_ID}.p8`, not a direct file path. The workflow was writing `$RUNNER_TEMP/AuthKey.p8` and passing that path. Fix: write to `$RUNNER_TEMP/private_keys/AuthKey_${KEY_ID}.p8` and pass the directory. Pushed fix (`469539f`). Also added `workflow_dispatch` trigger to `ios.yml` so it can be re-triggered manually. ## Lessons Learned

- Fork (`loveandrobots/olivia`) does not have Actions enabled. All CI runs happen on upstream (`LoveAndCoding/olivia`). PRs for deployable changes must go against upstream.
- GitHub Actions `push` trigger with `paths` filter won't fire on workflow-only changes. Remove paths filter for deploy triggers if you want every merge to deploy.
