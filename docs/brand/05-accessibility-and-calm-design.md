# 05 · Accessibility & Calm Design

## Designed for real minds.

---

Olivia is built for people whose minds work differently — and that means accessibility isn't an afterthought, it's the foundation. These principles should guide every design decision, from color choices to notification behavior to information density.

### Reduce visual noise

Limit the number of distinct elements visible at any time. Use generous whitespace. Avoid competing visual hierarchies. One primary action per screen.

### No guilt mechanics

Never use streak counters, overdue badges, red notification dots, or any pattern that punishes the user for not engaging. The app should welcome you back the same way whether it's been an hour or a month.

### Progressive disclosure

Show only what's needed right now. Details are available but never forced. Let users drill into complexity when they choose to — don't front-load it.

### Gentle motion only

If animation is used, it should be slow, purposeful, and never attention-grabbing. No bouncing, pulsing, or shaking. Respect reduced-motion preferences at the OS level.

### Forgiving interactions

Every action should be undoable. Every input should be flexible (accept multiple date formats, don't require specific phrasing). Never punish imprecision.

### Consistent, predictable patterns

Neurodiverse users benefit from consistency. Elements should behave the same way every time. Navigation should be predictable. Avoid layout shifts and surprise UI changes.
